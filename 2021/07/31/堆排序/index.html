堆排序

堆排序

原理：

用要排序的元素生成二叉大堆，利用二叉大堆特性，当前最大的元素就会出现在堆顶，保存当前最大值，去除当前堆顶元素，使用剩余的元素生成新的二叉大堆。n 个元素，经过 n -1 次这样的操作后，我们每一次的堆顶都是当前的最大值，这些堆顶的出现顺序就是我们要的排序顺序。

二叉堆： 一棵完全二叉树 （除最后一层，每一层都是满的）。 二叉堆又分为大堆和小堆。大堆的特性要求父结点的值一定要大于等于子结点的值。小堆则要求父结点的值一定小于等于子结点。

流程

从上面的流程来看，第一步要做的事就是用当前要排序的元素构建一颗二叉树。

构建二叉树通用的方法可以采用二叉树的结构体

而我们现在需要构建的是一个比较特殊的完全二叉树，由于其特性，我们可以采用一种更简单的方式。 利用二叉树结点跟数组下标的对应关系, 我们就可以直接把一个数组看成一棵完全二叉树。

二叉树已经有了，现在要做的事情就是要让其成为大顶堆 （每一个父结点的值一定要大于等于子结点的值），也就是 a[i] >= max (a[ 2 * i + 1], a[ 2 *i + 2] ).

流程: 比较父结点与两个子结点的大小，如果父节点小于子结点中其较大的一个，则交换这两个值, 使其满足大堆顶特性。，如果元素发生了交换，对发生交换的子结点执行相同，以保证交换后的以子结点为根的子树继续满足大顶堆特性。

从下到上，对每一个父节点进行max-heap操作，得到一个人完整的大顶堆。

优化：

​ 其实没有必要从最后一个结束开始进行max-heap操作，我们只要从有子树的最后一个父节点开始就可以。

​ 根据完全二叉树的特性，这个大概位置就是 L max-index / 2⌋。如果元素的个数正好为偶数个。最后一个父结点的位置正好是L max-index / 2⌋，如果是奇数个，L max-index / 2⌋则是最后父结点的下一个位置。并不是很精确。但起码比每次从最后一个开始要节省一些时间。如果想要计算最后一个父结点的精确位置也并不是一件难事。

二叉顶大堆已构建完成，接下来就是对堆顶进行操作，达到排序目的。

为了不再创建新的数组，节省内存，我们给build_max_heap(a) max-heap(a, i)和增加一个参数，用来表示要操作数组的长度范围。这样我们就可以用这个长度来排除已排好序的元素，对剩余的元素执行操作。
