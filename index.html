快速排序

原理： 把数字A[p..r]划分成两个子数组 A[p..q -1 ] 和 A[q + 1 .. r], 使得 A[p..q -1 ]中的值都小于 A[q], A[q + 1 .. r]中的值都大于A[q], 递归调用分割方法，直至不能再分割。

第一步实现数组的分割

依次用数组中的值与分界值进行比较，如果比分界值小，把它与待替换的位置交换，然后待替换向后移动。 直到遍历完成，然后交换分界值与最后一次替换的下一个位置。

分界值： 数组的最后一个元素

待替换： 初始值为 -1， 意味着当前不需要替换

如上图所示， 首先用 4 和 5 比较， 由于第一个元素 5 > 4, 不做任何操作，只是把i向后移一位。 接下来比较 2 和 4，由于 2 < 4, 根据规则，需要交换，

需要替换的位置 cp 向前 移动 一位 + 1，变成0，交换 a[i]和 a[cp]的值。依次执行，至到遍历完数组中除分界值的所有元素。然后交换分界4与cp的值5

最后形成 2 1 3 2 4 6 7 5， 这样以4为分界点，4前面的都比4小，4后面的都比4大。

第二步递归数组的分割

当进行一次第一步的数组分割时，我们会第到一个分割点，分割点前后分成两个子数组。我们对两个子数组执行相同的操作。这样就形成了快速排序的递归操作。

堆排序

堆排序

原理：

用要排序的元素生成二叉大堆，利用二叉大堆特性，当前最大的元素就会出现在堆顶，保存当前最大值，去除当前堆顶元素，使用剩余的元素生成新的二叉大堆。n 个元素，经过 n -1 次这样的操作后，我们每一次的堆顶都是当前的最大值，这些堆顶的出现顺序就是我们要的排序顺序。

二叉堆： 一棵完全二叉树 （除最后一层，每一层都是满的）。 二叉堆又分为大堆和小堆。大堆的特性要求父结点的值一定要大于等于子结点的值。小堆则要求父结点的值一定小于等于子结点。

流程

从上面的流程来看，第一步要做的事就是用当前要排序的元素构建一颗二叉树。

构建二叉树通用的方法可以采用二叉树的结构体

而我们现在需要构建的是一个比较特殊的完全二叉树，由于其特性，我们可以采用一种更简单的方式。 利用二叉树结点跟数组下标的对应关系, 我们就可以直接把一个数组看成一棵完全二叉树。

二叉树已经有了，现在要做的事情就是要让其成为大顶堆 （每一个父结点的值一定要大于等于子结点的值），也就是 a[i] >= max (a[ 2 * i + 1], a[ 2 *i + 2] ).

流程: 比较父结点与两个子结点的大小，如果父节点小于子结点中其较大的一个，则交换这两个值, 使其满足大堆顶特性。，如果元素发生了交换，对发生交换的子结点执行相同，以保证交换后的以子结点为根的子树继续满足大顶堆特性。

从下到上，对每一个父节点进行max-heap操作，得到一个人完整的大顶堆。

优化：

​ 其实没有必要从最后一个结束开始进行max-heap操作，我们只要从有子树的最后一个父节点开始就可以。

​ 根据完全二叉树的特性，这个大概位置就是 L max-index / 2⌋。如果元素的个数正好为偶数个。最后一个父结点的位置正好是L max-index / 2⌋，如果是奇数个，L max-index / 2⌋则是最后父结点的下一个位置。并不是很精确。但起码比每次从最后一个开始要节省一些时间。如果想要计算最后一个父结点的精确位置也并不是一件难事。

二叉顶大堆已构建完成，接下来就是对堆顶进行操作，达到排序目的。

为了不再创建新的数组，节省内存，我们给build_max_heap(a) max-heap(a, i)和增加一个参数，用来表示要操作数组的长度范围。这样我们就可以用这个长度来排除已排好序的元素，对剩余的元素执行操作。

归并排序

原理：

递归的合并N个已排序的子序列.

如上图所示，我们把单个的数字看成是一个已排序的子序列，然后不停的两两合并已排好序的子序列，直接全部合并完成。

流程

  1. 橙色部分为分解过程 2.黄色为合并过程 3. 绿色为执行顺序

从上面的流程图来看，这是一个标准的递归式解决方法。归并排序的每一层都包含三个步骤

  1. 分解 : 将 n 个 元素 分割 成含有 n / 2个元素的子序列
  2. 解决： 用归并排序法对两个子序列递归的排序
  3. 合并： 合并两个已排序的子序列得到排序的结果

我们可以把递归形象的理解成 [ 领导-员工 ] 任务模式。领导把任务分解成多个子任务然后派发给自己的多个直接下属， 然后下属再把子任务拆分派个自己的下属，下属的下属完成任务后把结果返回给下属，下属把结果处理后返回给领导，领导对返回的结果处理。

例如领导A要把 5 2 4 7 四个数字排序，

  1. 他把这个任务分成两个子任务 【5， 2】 【4，7】 分别交给自己的下属 A-B, A-C。
  2. A-B, A-C拿到任务后发现任务还可以接续分解， A-B 把自己的任务又分成 【5】【2】分配给 B-D 和 B-E。
  3. B-D和B-E拿到任务后发现自己的任务无法分解了， 自己进行处理 （单个数字的序列我们可以理解为已经排好序了). B-D 和 B-E ，把处理完的结果 【5】和 【2】返回给 A-B， A-B对结果进行处理得到 【2， 5】
  4. A-C 把自己的任务又分成 【4】【7】分配给 C-F 和 C-G。
  5. C-F和C-G拿到任务后发现自己的任务无法分解了， 自己进行处理 （单个数字的序列我们可以理解为已经排好序了). C-F 和 C-G ，把处理完的结果 【4】和 【7】返回给 A-C， A-C对结果进行处理得到 【4， 7】
  6. A-B和A-C把得到的结果 【2， 5】和 【4， 7】 返回给A， A对拿到的结果进行处理得到 【2， 4，5，7】

我们需要一个用来合并两个子序列的方法 merge

原理： 依次比较两个已排好序的子序列的每一个元素, 选择符合条件的一个放入对应的位置。

这只是为了实现算法采用的一种方式。没有这个也完全可以。它的作用就是用来防止两个子序列的游标 i 和 j 越界。如上面代码所示，当一个子序列的元素全部被取完后，它的游标会指向正无穷大，意味着这个序列没有元素了，所以接下来的每一次比较他都是大的那个，这样就会把另一个数组剩余的元素放到指定的位置。如果是数组从大到小的排序，这个值就需要是负无穷大了。
